Projects

Networked Camera

Spring 2016

Two classmates and I worked with Altera 2 developement boards to simulate a remote camera. We were tasked with linking the two boards through ethernet and sending instructions from a one to another using software-defined network layers that we created in C++ and Verilog. A 'master' board would sent commands to a 'slave' board to take an image, manipulate it in one of several ways, then compress and send the image back to the master. We designed the entirety of the system and the TCP protocol used to transmit and achknowledge the packets of data that the slave and master sent. We were judged by the effectiveness of our compression and the robustness of our system.

General MIDI Explorer

Fall 2015

Our Computer Systems Lab was split into random groups of three with the task fo creating a general purpose MIDI explorer with an Atmel Atmega32 AVR chip and C code. Our team met frequently and divied the hardware, software and testing requirements among our members. By using the datasheet for the chip and our compression algorithm, we were able to store up to 1KB of audio on the chip's memory and play back the original recording. For extra credit, we added a photodiode that changed the way that the audio was played back depending on the light hitting it. The project involved debugging the code using gcc debugger, the hardware using a multimeter, and the signals using a logic analyzer.

UNIX Shell Emulator

Fall 2015

Worked as a pair to create a UNIX shell in C++. Used fork() and exec() functions to execute the desired commands. Our end result was able to execute all UNIX commands in /bin, built-in commands (cd, jobs, and exit), I/O redirection, pipelining, and background processes in single or batch modes. We also wrote documentation to describe the functionality of our shell.

MIPS Processor Simulator

Fall 2015

I led a team of four to create a simulation of a pipelined MIPS processor in C. The design of the code we wrote was based on the physical layout of a real processor and we used a collaborative coding environment to distribute the workload among the group. Together we created a program that maintained the correct register data at each stage of the instruction set. The simulator could be run in single step and batch modes and had the ability to handle hazards presented to it.
